# Error patterns & drill design (telc B1)

Use for `[compare exams]` and `[weaknesses]`, and after every grading. This is a **living
profile** — the seed patterns below come from the user's practice history; always merge in
the latest from memory and from the exam just graded, and re-rank by how much each pattern
is currently costing.

## Seed weakness profile (update from memory each session)

Ranked by damage (biggest point-loss first):

1. **Leseverstehen T3 — X-overuse.** Marking `x` (no match) on situations that *did* have a
   matching ad. The single most damaging section failure historically. **Fix:** default to
   finding a match before choosing X; only X after checking all ads against every hard
   requirement.
2. **Hörverstehen +/− traps (T2 especially).** The recurring **setup→twist reversal**
   (classically around Q51) plus negation/qualifier misses. Most consistent recurring error
   across exams. **Fix:** decide only after the statement's clause finishes; flag
   negations/qualifiers in the Lesepause. See the trap taxonomy in `section-strategies.md`.
3. **Sprachbausteine T2 — connectors & collocations.** Consistently the weakest written
   section. **Fix:** memorise the email/letter stock phrases + top collocations, and the
   connector-logic-then-word-order routine.
4. **Sprachbausteine T1 — preposition selection; reflexive-verb Perfekt (`hat sich …` vs.
   `wurde`); singular/plural.** **Fix:** learn verb+preposition as chunks; drill reflexive
   Perfekt forms.
5. **Leseverstehen T2 — comparison-direction and main-point-vs-side-detail traps.**
6. **Leseverstehen T1 — synonym/detail traps in headline matching.**

Strengths to protect (don't over-drill these): Hörverstehen T1/T3 are often strong;
Sprachbausteine T1 is sometimes strong; some exams show very high SB overall. Keep the
profile honest — promote/demote patterns as new results come in.

## `[compare exams]` — what to produce
1. Pull each logged exam's per-Teil scores from memory/logs.
2. Surface **recurring question types and traps** (which Teil leaks points across multiple
   exams, not just once).
3. Flag **question-bank reuse** (shared texts/passages).
4. Recommend the 2–3 highest-leverage drill targets and offer to build them via
   `[practice]`.

## `[weaknesses]` — what to do (also auto-runs after grading)
1. **Record** the just-graded result's strengths and failures to memory, phrased as
   *patterns* (e.g. "Hören T2 setup→twist reversal", "Lesen T3 X-overuse"), not one-offs.
2. **Re-rank** the living profile by current damage.
3. **Design targeted drills** (recipe below) for the top 2–3 patterns and offer to run them.

## Drill-design recipe (turn a failure into practice)
For each weak pattern, build a short set that:
- **Re-asks the exact missed items** from the graded exam (same question, so the user meets
  the trap again knowingly), then
- **Generates 3–5 similar new items** in the same trap family at B1 level, then
- **Corrects** each with the rule and a **B1-safe alternative**, naming the trap type.

Trap-family templates:
- **Hören +/− (setup→twist):** write a 2–3 line mini-script that states X then reverses it
  after `aber/doch/eigentlich`; ask R/F; show why the last clause decides.
- **Hören +/− (negation/qualifier):** contrast `alle` vs `die meisten`, `erst am Freitag`
  vs `am Freitag`; ask R/F.
- **Lesen T3 (X-overuse):** give a situation with a genuine-but-terse matching ad plus a
  near-miss decoy; make the user justify match-vs-X against each hard requirement.
- **SB T2 (connector):** one gap, four connectors from different logic classes; make the
  user state the logical relation first, then pick the one whose word order fits.
- **SB T1 (preposition/reflexive):** verb+preposition chunk gaps; reflexive Perfekt gaps
  (`hat sich gefreut`).

Keep drills short and correction-heavy. After a drill, update memory with what improved and
what's still leaking, so the next session targets the right thing.
