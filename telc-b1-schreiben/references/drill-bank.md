# Drill bank — targeted, personalised written-error drills (telc B1)

This file powers `/written-drill`. The whole point is that the drill is **not
generic** — it is built from the user's own tracked mistakes. The coach already
logs the user's recurring written errors to memory after every `/written-grade`
(see the self-improvement section of SKILL.md). `/written-drill` reads that log and
turns it into a **challenging, mixed test** that hammers exactly the things the user
keeps getting wrong — from the Betreff down to spelling — and nothing they've
already mastered.

Explanations in English; all German at A2–B1 level. Strict, correction-heavy, no
filler. The drill must feel like a real test, not a warm-up: withhold answers until
the user has attempted everything, then grade hard.

---

## Step 1 — Read the user's leak map before writing a single item

Before generating a drill, assemble what the user actually gets wrong:

1. **Memory** — the recurring-written-errors note (the "SCHREIBEN — recurring
   written-exam errors …" memory). This is the primary source. It lists the user's
   confirmed leak categories, their **strengths**, and which patterns have been
   **retired/fixed**.
2. **This chat** — any mistakes from letters graded earlier in the current
   conversation. Fresh evidence outranks stale notes.
3. If both are thin (brand-new user, no history), fall back to the general telc-B1
   written-error taxonomy in `common-mistakes.md` and build a broad diagnostic drill
   across all categories below.

From that, sort every category into three buckets — this drives the weighting:

- **HOT** = still recurring / never confirmed fixed → the bulk of the drill.
- **WATCH** = general telc-B1 traps the user hasn't clearly failed *or* passed →
  a few items, to keep coverage comprehensive (the user explicitly wants "everything").
- **MAINTENANCE** = recently confirmed fixed → 1 item max, as a spot-check; if they
  ace it again, tell them and stop testing it.

Never silently drop a category the user asked to cover. If they say "test everything,"
touch every HOT and WATCH category at least once.

## Step 2 — The error taxonomy (the categories a drill can target)

Each category lists **what the user's specific leak tends to be** (adapt to the
actual memory note — these are the known ones, not a fixed list) and item ideas.

### A. Betreff & framing (Anrede · Datum · Betreff · Schluss)
Leak: **skips the Betreff when the task explicitly asks for one**; sometimes abrupt
opening/closing. Free Kriterium-II points lost.
- Give a one-line scenario → user writes a correct **Betreff + Anrede + first
  sentence (Einleitung)** in the right register.
- Give a letter with the Betreff missing → user supplies a fitting one.
- Show 3 Betreff options for a scenario → user picks the appropriate register + says why.

### B. Reflexive verbs (reflexive pronoun present & correct)
Leak: **omits the reflexive pronoun** — *sich freuen* (mich/dir), *sich treffen*
(uns), *sich Sorgen machen* (dir). Also the fixed plural **Sorgen** (not *Sorge*).
- Gap-fill the reflexive pronoun: `Ich freue ___ auf …`, `Wann treffen wir ___?`,
  `Mach ___ keine Sorgen.`
- Error-fix: `Ich habe sehr gefreut, dass …` → insert **mich**; `wenn wir treffen` →
  **uns**.
- Perfekt trap: `Ich bin sehr gefreut` → *sich freuen* takes **haben + mich**
  (`Ich habe mich gefreut`), not *sein*.

### C. Verb position (the #1 point-killer — see common-mistakes.md)
Leak: **modal/finite verb not moved to the end** in subordinate & relative clauses
(`die du kannst folgen` → `der du folgen kannst`); occasionally verb not in 2nd
position after a fronted time phrase.
- Rearrange: give scrambled clause → user writes it with the verb in the right slot.
- Complete the clause: `Ich schicke dir einen Plan, dem du …` (folgen / können) →
  `… folgen kannst`.
- Fronted-adverbial fix: `Morgen ich fahre nach Berlin.` → `Morgen fahre ich …`.
- weil/dass/wenn to the end: `…, weil ich bin krank` → `…, weil ich krank bin`.

### D. Case after verbs (dative vs. accusative object)
Leak: **dir/dich confusion** — *erklären + dative* (`erkläre dich` → `erkläre dir`);
dative verbs (helfen, danken, gefallen, gehören, antworten).
- Gap-fill: `Ich erkläre ___ (du) alles.`, `Das Café gefällt ___ (du) bestimmt.`,
  `Ich helfe ___ (du) gern.`
- Error-fix: `Ich danke dich für …` → **dir**.
- Two-object item: `Ich schicke ___ (du) ___ (der Plan).` → **dir den Plan** (dat + acc).

### E. Case after prepositions (der/dem/den; two-way prepositions)
Leak: general der/dem/den slips. *Tolerable* on the exam if meaning survives, but worth
tightening. Include some **Wechselpräpositionen** (in/auf/an → Akk for motion, Dat for
location).
- Gap-fill the article: `Wir gehen in ___ Wüste.` (Akk → **die**),
  `Wir treffen uns in ___ Café.` (Dat → **dem/im**).
- Error-fix: `mit den Zug` → **dem**; `Ich freue mich auf der Reise` → **die Reise**.
- Fixed collocation: `sich Sorgen machen **um**`, `sich freuen **auf/über**`,
  `warten **auf** + Akk`, `denken **an** + Akk`.

### F. Verb conjugation (present tense, esp. du-forms & irregulars)
Leak: **du-form errors** — `du muss` → `du musst`, `kanst` → `kannst`; stem-changers
(fahren→du fährst, geben→du gibst, nehmen→du nimmst).
- Conjugate-in-context: `___ (müssen, du) jeden Tag ins Büro?`,
  `___ (können, du) von zu Hause arbeiten?`
- Error-fix: `Was macht du?` → **machst**; `du fahrt` → **fährst**.
- Modal-set spot check: mix müssen/können/wollen/dürfen du-forms in one item.

### G. Fixed-phrase constructions (the darauf/dass family)
Leak: **drops *dass* or the *dar-* prefix** in *freuen* constructions —
`freue mich, darauf du …` (→ *dass du*), `freue mich auf, dich zu …`
(→ *darauf, dich zu …*).
- Two-way choice: `Ich freue mich, ___ du kommst.` (dass) vs.
  `Ich freue mich ___, dich zu sehen.` (darauf).
- Build the sentence from cues: *(freuen / du / kommen)* → `Ich freue mich, dass du
  kommst.`; *(freuen / dich wiedersehen)* → `Ich freue mich darauf, dich
  wiederzusehen.`
- Rule to reinforce: **freuen + dass + full clause**; **freuen + darauf + zu-clause /
  auf + noun**.

### H. Spelling & capitalization
Leak: **noun capitalization** (wetter→Wetter, wüste→Wüste, monate not Monaten),
ei/ie (errinerungin→Erinnerungen), double consonants, foreign-plural endings.
- Correct the sentence: `ich habe viele errinerungin an das wetter.` → capitalise +
  spell.
- Noun-hunt: give a sentence, user lists which words must be capital.
- Plural check: `zwei Monaten` (after a number, no dative-n) → **zwei Monate**.

### I. Compound nouns (write them as one word)
Leak: **splits compounds** — `Wandern Schuhe` → `Wanderschuhe`.
- Join: `Wander + Schuhe`, `Lieblings + Café`, `Reise + Pass`, `Arbeits + Platz`.
- Error-fix: `mein Lieblings Café` → **Lieblingscafé**.

### J. Connectors & structure (Kriterium II)
Leak: **abrupt, unlinked sentences** ("stub" style) and occasional jumpy ordering.
- Join two stubs with the right connector: *(Ich komme im März. Es ist Frühling.)* →
  `Ich komme im März, weil es Frühling ist.`
- Insert a connector to smooth a hard cut (`… . Jordanien ist noch kalt.` → add
  *Übrigens / Allerdings* or fold into the previous idea).
- Order task: give the 4 Leitpunkte shuffled → user proposes a sensible order + a
  one-line reason.

## Step 3 — Build the drill (make it genuinely challenging)

The user asked for a **challenging** test, not flashcards. Design accordingly:

**Composition**
- **10–15 items** total, grouped into **4–6 short sections by category**, HOT
  categories carrying the most items. Label each section (e.g. *A. Reflexivverben*,
  *B. Verbstellung*).
- **Mix the formats** (gap-fill, error-fix, sentence-build, transformation, the mini
  structural task). Don't make it all gap-fills — varied formats test real command,
  not pattern-matching.
- **End with one applied mini-write** (2–4 sentences on a telc topic) that *forces
  several HOT patterns at once* — e.g. "Write a short closing to a friend that (a)
  uses *sich freuen* correctly, (b) has one weil-clause with the verb at the end, and
  (c) includes a correct Betreff line above it." This is where isolated skills have to
  survive together, like in the real exam.

**Difficulty ramp** (unless the user requests `easy`/`hard`):
- Start each section with a clean single-error item, then escalate to items that
  **stack two leak types** (e.g. a sentence that's wrong on both reflexive pronoun
  *and* verb position), and hide a **distractor** (a plausible-looking but correct
  part they might "fix" by mistake). Stacking mirrors how errors actually cluster in
  the user's letters.
- If the user passes a category cleanly last time, its items should be **harder**
  this time (stem-changing verbs, two-object dative+accusative, Wechselpräposition
  with motion/location contrast).

**Optional argument** — `/written-drill [category | easy | hard | quick]`:
- a named category (`reflexive`, `verb position`, `cases`, `Betreff`, `spelling`,
  `connectors`, …) → drill only that, deeper.
- `quick` → 5–6 items, HOT categories only.
- `easy` / `hard` → shift the whole ramp down/up.
- no argument → full personalised mixed drill across HOT + WATCH.

**Presentation rules**
- Number every item. State the instruction per section in one short line.
- **Do NOT reveal answers, hints, or the target rule inside the drill.** No "(dative!)"
  next to a gap. The challenge is spotting it unaided.
- Tell the user to answer **all** items in one go, then you'll grade. Don't grade
  item-by-item as they trickle in unless they ask.
- Keep it copy-pasteable and clean — this is a test sheet.

## Step 4 — Grade the drill (strict, itemised, and update the leak map)

When the user submits answers, grade **every item**:

1. **Per item:** their answer → the correct answer → the **one-line rule** behind it.
   Mark ✅/❌. Be exact; don't wave through a near-miss.
2. **Per category (section) score**, e.g. *Reflexivverben 3/3 ✅ · Verbstellung 1/3*.
   This shows the user which leaks are closing and which are still open — the whole
   value of a tracked drill.
3. **Headline:** how many of the HOT leaks held up under pressure, and the single
   biggest remaining weakness to attack next.

Then **close the self-improvement loop** (this is mandatory — it's what makes the
system compound):

- **Retire/soften** any category the user now aces (especially a MAINTENANCE one):
  update the memory note to mark it fixed, and tell the user it's improved.
- **Escalate** any HOT category they still fail: keep it flagged, and make it the
  opening drill next time.
- **Add** any *new* error pattern that surfaced in the drill to the memory note.
- Keep the memory note current and de-duplicated (update, don't append duplicates).
  Store skill-pattern notes only — never sensitive personal data, never verbatim
  command dumps.

Finish with a short, motivating **"your leak map now"** line (what's green, what's
still red) and a concrete next step (another drill on the red category, or
`/written-examine` to prove it in a full letter).

## Worked micro-example (shape only — always regenerate fresh, personalised items)

> **Section A — Reflexivverben.** Setzen Sie das richtige Reflexivpronomen ein.
> 1. Ich habe ___ sehr über deinen Brief gefreut.
> 2. Wann treffen wir ___ nächste Woche?
>
> **Section B — Verbstellung.** Vervollständigen Sie den Satz (Verb ans richtige Ende).
> 3. Ich schicke dir einen Plan, ___ (dem / du / folgen / können).
> 4. …, weil ich viele Probleme mit der Arbeit ___ (haben, Perfekt).
>
> **Section C — Mini-Schreiben.** Schreiben Sie 2 Sätze als Abschluss eines Briefs an
> eine Freundin: benutzen Sie *sich freuen* korrekt **und** einen weil-Satz mit dem
> Verb am Ende. Schreiben Sie darüber auch eine passende Betreff-Zeile.

Grade example for item 3: answer `dem du folgen kannst` ✅ — *in a relative clause the
modal verb goes to the very end*. Answer `dem du kannst folgen` ❌ — same rule, verb
misplaced (this is the user's classic leak; flag it if it repeats).
